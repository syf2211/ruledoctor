## 1. 小红书

**标题（选一）：**

- Claude 又不看 CLAUDE.md？装这个 Skill 再新开对话
- 给 AI 编程助手装一个「规则守门员」开源 Skill
- 规则写了等于没写？让 Agent 先读再改代码

**正文：**

你是不是也写过 `.cursorrules` / `CLAUDE.md`，结果 AI 照样乱改、照样该跑的测试不跑？

更难受的是：它还会说「已按规范完成」——你根本不知道是真是假。

我开源了一个 **RuleDoctor** Agent Skill，给 Claude Code / Codex / Cursor / OpenCode 用：

✅ 新开对话会先 **Read 项目规则**  
✅ 默认告诉你：读了哪些文件 + 本场几条硬规矩（不长篇念经）  
✅ 深层文档可配 `required_reads`，不会漏读  
✅ 想较真时可选「检测报告」，揪假遵守  
✅ 危险 `git push --force` 还可选配硬拦截  

**安装（复制）：**

```text
npx skills add syf2211/ruledoctor@ruledoctor -g -y
```

装完务必 **关掉旧对话，新开一场**，在有规则文件的项目里试。

## 2. 知乎 / CSDN 长文

**标题：** 写了 CLAUDE.md 为什么 AI 还是乱改？我做了个 Skill 让它先读规则再动手

**摘要：** 项目规则经常没进上下文，或被压缩后遗忘。RuleDoctor 是可安装的 Agent Skill，配合可选检测工具，让「遵守规则」从凭感觉变成可验证。

**正文结构：**

1. **痛点**：规则静默失效；助手口头答应、代码不照做；长对话后忘禁忌。  
2. **常见误区**：写更长 system prompt ≠ 助手会读仓库里的 CLAUDE.md。  
3. **RuleDoctor 是什么**：开源 Skill（`skills/ruledoctor/SKILL.md`），一条 `npx skills add` 安装。  
4. **用起来会怎样**：开场先读规则；`required_reads` 强制必读清单；两层汇报避免刷屏。  
5. **和「只写规则」的区别**：Skill 改当场行为；可选 CLI 做事后核对；可选 Hook 拦危险 shell。  
6. **安装步骤**：命令 + 新开对话 + 如何算生效。  
7. **诚实边界**：「必须用中文、必须写验证说明」主要靠 Skill，不能靠 shell Hook 硬拦回复。  
8. **链接与贡献**：GitHub、skills.sh、Issue 欢迎反馈。

**CSDN 标签：** Claude、Cursor、Agent、开源、AI编程
