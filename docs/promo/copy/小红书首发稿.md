## 标题

Claude 又不看 CLAUDE.md？我做了个开源 Skill

## 正文

你有没有遇到过这种情况：

项目里明明写了 `CLAUDE.md`、`.cursorrules`、`AGENTS.md`，AI 编程助手还是直接改代码。更麻烦的是，它还会说“已按规范完成”，但你根本不知道它到底读没读规则。

我开源了一个小工具：**RuleDoctor**。

它首先是一个 Agent Skill，给 Claude Code / Codex / Cursor / OpenCode 用。装好并新开对话后，助手应该先读取项目规则和你配置的 `required_reads`，再开始改代码。

它做的事很简单：

- 先读项目规则，再动手改仓库
- 默认只告诉你读了哪些文件 + 最多 3 条硬约束
- 上下文被压缩后，提醒助手重新读规则
- 遇到 `git push --force` 这类危险命令时应该拒绝
- 想较真时，还可以用可选 CLI / Hook 做事后报告和命令拦截

安装命令：

```text
npx skills add syf2211/ruledoctor@ruledoctor -g -a codex -y
```

Claude Code 用 `-a claude-code`，Cursor 用 `-a cursor`。

装完记得关掉旧对话，**新开一场**，在有规则文件的项目里试一个小任务。

GitHub 搜：`syf2211/ruledoctor`

如果你也被“规则写了但 Agent 不看”坑过，欢迎 Star / Issue，我会继续把 Hook、报告和更多 Agent 适配补全。
