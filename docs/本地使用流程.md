# 本地使用流程（Claude Code / 桌面版）

用 **10 分钟**在本机证明 RuleDoctor 有用。路径默认在：

`/Users/syf/Desktop/RuleDoctor工作区/rule-doctor`

---

## 你能得到什么

1. 项目里自动有 **CLAUDE.md**（没有就生成）和 **`.ruledoctor.json`**（至少禁止 force push）。
2. **Claude Code 每次会话结束**自动跑体检，写出  
   **`<项目>/.ruledoctor/last-report.html`**，macOS 会自动 `open` 打开。
3. 也可手动：`ruledoctor --last-session` 扫**最近一次**会话。

---

## A. 对 Agent 说（推荐）

在 **Claude Code** 里，先 `cd` 到你的项目目录，然后发送：

```text
请在本项目执行 RuleDoctor 安装：
1) cd 到 /Users/syf/Desktop/RuleDoctor工作区/rule-doctor 运行 npm run build
2) node dist/index.js setup -p <我的项目绝对路径>
3) 用 node dist/index.js --cwd <我的项目> --last-session 试跑，把终端摘要贴给我
```

把 `<我的项目绝对路径>` 换成你真实在写的文件夹（例如桌面上的某个仓库）。

---

## B. 自己敲（3 条命令）

```bash
# 1. 构建引擎（只需一次）
cd "/Users/syf/Desktop/RuleDoctor工作区/rule-doctor"
npm install && npm run build

# 2. 在你的项目里安装（示例：给 rule-doctor 自己装一遍）
node dist/index.js setup -p "/Users/syf/Desktop/RuleDoctor工作区/rule-doctor"

# 3. 用最近一次 Claude 会话试跑
node dist/index.js --cwd "/Users/syf/Desktop/RuleDoctor工作区/rule-doctor" --last-session
```

`setup` 会：

- 没有规则文件 → 写 **CLAUDE.md**
- 没有检查配置 → 写 **`.ruledoctor.json`**
- 写 **`.claude/settings.json`** 里的 **SessionEnd** Hook

---

## C. 证明「有用」的标准动作

1. **打开 Claude Code**（或桌面版，需能写 `~/.claude/projects/...` 日志），在**已 setup 的项目目录**里做一小件事，例如：
   - 「请读 CLAUDE.md，然后故意在回复里提一下 `git push --force` 作为反例。」
2. **结束会话**（或开新会话触发 SessionEnd）。
3. 看是否自动弹出 **`last-report.html`**：
   - **R 类规则**里 force push 相关应为 **遵守 ✗**（若会话里出现了该命令）。
   - 总分旁应显示 **「仅读到率」或完整分**（已配置 checker 后才是完整遵守率）。

若没有自动弹窗：

```bash
open "<项目>/.ruledoctor/last-report.html"
```

---

## D. Codex / 其他 CLI

当前只保证 **Claude Code 的 JSONL** 格式。Codex 若日志路径不同，先用：

```bash
node dist/index.js --cwd <项目> --session <你的 codex 会话文件路径>
```

把路径问清楚后再做 adapter（后续细化）。

---

## E. 桌面版 Claude（非 Code）

- 会话仍在 `~/.claude/projects/.../*.jsonl`，**可以**用 `--session` 指向具体文件。
- **SessionEnd Hook** 是否触发取决于桌面版是否执行 `.claude/settings.json` hooks — 若未触发，用 **C 节手动 `--last-session`** 同样能验证。

你之前的 **「AI coding tools pain points」** 会话文件：

`~/.claude/projects/-Users-syf-Downloads/f9df071c-de89-492b-9479-f3285f1567d7.jsonl`

可用：

```bash
node dist/index.js setup -p /Users/syf/Downloads
node dist/index.js --cwd /Users/syf/Downloads --rules docs/examples/pain-points-research-rules.md \
  --session "$HOME/.claude/projects/-Users-syf-Downloads/f9df071c-de89-492b-9479-f3285f1567d7.jsonl"
```

（Downloads 下建议先 `setup` 生成 CLAUDE.md，再换成你自己的规则文件。）

## 计分说明（避免误解）

| 情况 | 显示 |
|------|------|
| 只有 CLAUDE.md，没有 `.ruledoctor.json` checker | **仅读到率**；遵守率为 **—**；总分 = 读到率 |
| 有 `.ruledoctor.json` | 读到率 40% + 遵守率 60% |
