1. 这个 demo 在模拟什么？

假想的「订单服务」项目：你在 .cursorrules 里写了 8 条硬性规则，AI 帮你写了 src/orders.ts，还在会话里执行了 git push --force。

代码里故意留的坑

  • 金额用浮点 99.5（规则要求整数「分」）
  • 日期 2026/06/16（规则要求 YYYY-MM-DD）
  • 使用了 any
  • 错误类名用了 ErrorXxx（这条反而遵守了）

会话日志里故意设计的「读率」故事

session.jsonl 里，只有部分规则出现在发给模型的 <system_context> 中（模拟「规则没全加载 / 被截断」）。 同时 tool 结果里出现 git push --force，用于检测「读到了却违反」。

2. 你怎么用（一条命令）

cd ruledoctor
npm install && npm run build
node dist/index.js \
  --cwd examples/demo-project \
  --session examples/demo-project/session.jsonl

# HTML 报告
node dist/index.js \
  --cwd examples/demo-project \
  --session examples/demo-project/session.jsonl \
  --format html --out docs/demo-report.html

# CI 门禁（低于 80 分则 exit 1）
node dist/index.js ... --min-score 80; echo exit:$?

4. 工具在算什么？

维度数据来源含义
读到率Claude Code 式 .jsonl 会话规则 distinctive 词是否出现在发给模型的语料里（离线、不调模型）
遵守率.ruledoctor.json 检查器 + 代码树 + 会话命令正则/命令类规则是否被违反
总分40% 读到 + 60% 遵守用于排序和 CI --min-score

本 demo 检查器配置

浮点金额、斜杠日期、any、ErrorXxx、push --force 共 5 项；未配置检查器的规则显示「—」。
