# 分数是什么意思？（给人看的，不是给日志看的）

很多人看到「读到率」「遵守率」「37/100」会懵。下面用**普通人能懂**的话说清楚。

---

## CLI 到底看了什么？

**它不会扫描你整个硬盘，也不会读 Agent 的「内心」。**

它只做两件事：

1. **读你指定的「规则文件」**（例如 `CLAUDE.md`、`.cursorrules`），拆成一条条短规则（R1、R2…）。
2. **读一场对话的本地记录**（Claude / Codex / Cursor 保存在你电脑上的 `.jsonl` 日志），把里面所有文字拼成一大段，去查每条规则里的关键词有没有出现。

所以：**规则从文件来，证据从会话日志来。**

---

## 什么叫「规则」？

在这里，**规则 = 你写在项目里的明文条目**，例如：

- `CLAUDE.md` 里的 bullet
- `.cursorrules` 里的段落
- 你用 `--rules` 手动指定的某个 `.md`

**不算规则的东西（CLI 默认不管）：**

- Skill 目录里别的技能的说明（除非你把那个 SKILL.md 当 `--rules` 传入）
- 系统提示词里你看不见的部分
- 你心里想的要求但没写进文件

---

## 什么叫「应用了规则」？CLI 能判断吗？

要分开说：

### A. Skill（改行为）

**「应用了规则」**在 Skill 场景里指：助手 **Read 了规则文件**、按规则 **拒绝坏命令**、上下文变短后 **重读**。

这是你在**聊天里能看到的**，不是百分比。

### B. CLI（事后核对）

CLI **不能**判断助手「心里有没有遵守」。

它只能算两类**很粗的指标**：

| 指标 | 人话 | 怎么来的 |
|------|------|----------|
| **读到率（单条）** | 这场对话记录里，有没有出现这条规则里的关键词 | 在 jsonl 全文里搜规则拆出来的词，达到阈值算「出现过」 |
| **遵守率（单条）** | 你有没有配置「可自动检查」的项，且本场没踩雷 | 仅当 `.ruledoctor.json` 里写了检查（如禁止出现 `push --force` 命令） |

---

## 「读到率 0%」到底什么意思？

**不是**「你电脑上没有规则文件」。

**也不是**「遵守率是 0%」。

它通常表示：

> **在这条对话的保存记录里，几乎找不到这条规则对应的关键词。**

可能原因（CLI **不能**替你区分是哪一种，只能提示你）：

1. 助手**没读**规则文件，规则从未进过对话；
2. 规则在系统里以别的方式注入，但**关键词和规则文件写法不一致**，搜不到；
3. 对话太长，**前半段有、后半段被压缩没了**（需结合 Skill 重读）；
4. 你扫的 `--rules` 文件**不是**这场对话实际该遵守的那份（扫错文件）。

**举例（泛化，不含个人路径）：**

- 规则写：「改代码前先读 `docs/CONTRIBUTING.md`」
- 整场 jsonl 里从没出现 `CONTRIBUTING` 相关词 → 这条 **读到率很低或 0%**
- 规则写：「用 `ssh macmini`」且对话里大量出现 `macmini` → 这条 **读到率可能很高**

---

## 总分怎么来的？

- 若**没有**在 `.ruledoctor.json` 里配置检查项：  
  **总分 ≈ 有多少条规则在日志里「像出现过」的比例**（报告会写「仅读到率」）。

- 若**有**配置检查项：  
  总分 = 读到率和遵守率加权（见 README / 开发者文档）。

**没有配置检查时，不会出现「遵守率 100%」**——避免误导。

---

## 「遵守率」和「读到率」别混

| 情况 | 报告可能长什么样 |
|------|------------------|
| 有规则文件，没配 `.ruledoctor.json` | 只有读到率，遵守列是「—」 |
| 配了禁止 force push | 若日志里出现过 `push --force`，对应条 **遵守 ✗** |
| 规则在日志里出现过，但仍 force push | **读到高 + 遵守 ✗**（说明「看见了还做错」） |
| 项目目录下根本没有规则文件 | CLI 会报错让你加文件或 `--rules`，不是「0%」 |

---

## 你凭什么信这个 CLI？

它**不调用大模型**，只做：

- 读文件 + 读 jsonl + 字符串匹配 + 你配置的简单检查（如命令里是否出现某 substring）。

所以：

- **优点**：同一输入，结果可重复；能发现「规则文字根本没出现在对话里」这类硬事实。
- **局限**：关键词匹配不等于「助手理解了」；扫错规则文件会得出无意义分数。

**Skill 负责让下一场更好；CLI 负责用上一场日志给你一张核对单。**

---

## 我自己怎么跑？

```bash
ruledoctor --cwd /你的项目 --last-session
# 或指定日志：
ruledoctor --cwd /你的项目 --session /path/to/session.jsonl --rules /你的项目/.cursorrules
```

看每一行 **读到** 和 **遵守（—/✓/✗）**，比只看一个总分更有用。
