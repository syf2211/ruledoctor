# RuleDoctor 使用说明（Cursor · Claude Code · Codex）

**一句话**：在项目里写好规则 → 用任意 AI 写代码 → 会话结束（或敲一条命令）→ 看同一份 HTML 报告，知道规则有没有被读到、有没有被违反。

---

## 1. 安装（做一次）

在 RuleDoctor 源码目录（或 `npm i -g ruledoctor` 装好后）：

```bash
cd /你的项目
npm run build   # 若用源码：先在 ruledoctor 仓库里 build
node /path/to/ruledoctor/dist/index.js setup -p .
```

`setup` 会：

- 没有规则文件时生成 **CLAUDE.md**（名字历史原因，**三个工具都会当项目说明读**）
- 生成 **`.ruledoctor.json`**（例如禁止 `git push --force`）
- **Claude Code**：写入 `.claude/settings.json` 的 SessionEnd Hook  
- **Cursor**：写入 `.cursor/hooks.json` 的 `sessionEnd` Hook  
- 两者都会在会话结束时跑体检，并把报告写到 **`.ruledoctor/last-report.html`**（macOS 会自动打开）

---

## 2. 三个工具分别怎么用

| 工具 | 你要做的 | 报告怎么来 |
|------|----------|------------|
| **Cursor** | 正常在本项目里 Agent 聊天；`setup` 后无需再记命令 | 会话结束 Hook 自动出报告 |
| **Claude Code** | 在项目目录里 `claude`；`setup` 后无需再记命令 | SessionEnd Hook 自动出报告 |
| **Codex** | 在项目目录里用 Codex 写代码 | 结束会话后执行一条：`ruledoctor --cwd . --last-session`（或全局 alias） |

**手动统一入口**（任意工具、任意时刻）：

```bash
ruledoctor --cwd /你的项目 --last-session
```

会自动在 **Claude / Codex / Cursor** 的本地会话记录里，按修改时间选**最新一条**和当前 `--cwd` 相关的日志。

查看能找到哪些会话：

```bash
ruledoctor doctor -p /你的项目
```

## 3. 规则从哪来

- 项目根目录常见文件：`CLAUDE.md`、`AGENTS.md`、`.cursorrules`、`.cursor/rules/*.md` 等（自动发现）
- 也可用：`ruledoctor --rules path/to/rules.md --cwd .`

**遵守检查**只在 **`.ruledoctor.json`** 里配置（正则、禁止命令等）。没配置时，总分**只算读到率**，不会假装「遵守 100%」。

---

## 4. 分数是什么意思

- **读到率**：规则里的关键词是否出现在**本场会话的完整 transcript** 里（离线扫描 jsonl，不调模型）。
- **遵守率**：仅对 `.ruledoctor.json` 里配置了的项做检查（例如是否出现过 `push --force`）。
- **总分**：有遵守配置时综合；没有时 = 仅读到率。

演示样例（固定 37 分）：`examples/demo-project` + `npm run demo`。

---

## 5. 报告在哪

- Hook 自动：`你的项目/.ruledoctor/last-report.html`
- 自己指定：`ruledoctor --cwd . --last-session --format html --out ./report.html`

---

## 6. 常见问题

**Q：我没有 CLAUDE.md 怎么办？**  
跑 `setup`，会生成模板；按项目改几条硬规矩即可。

**Q：Cursor 工作区路径和文件夹名不一致？**  
`--last-session` 会在 transcript 里匹配你的 `--cwd` 路径，不限于文件夹编码名。

**Q：这和「让模型自觉遵守」一样吗？**  
不一样。RuleDoctor 是**事后体检 + 可配置拦截（后续版本）**，用日志证明规则有没有进上下文、有没有踩红线。
