## 1. 痛点贴：CLAUDE.md 不生效

**标题：** CLAUDE.md 写了，Claude Code 为什么还是乱改？

**正文：**

很多人以为项目里有 `CLAUDE.md`，Claude Code 就一定会先读。

实际不是这样。长对话、上下文压缩、规则文件太深，都会让规则像没存在过一样。

我做了一个开源 Skill：**RuleDoctor**。

它的作用很简单：新开对话后，先让 Agent 读项目规则，再开始改代码。默认只报：

- 读了哪些规则文件
- 本场最多 3 条硬约束
- 遇到 force push 这类危险操作应拒绝

安装：

```text
npx skills add syf2211/ruledoctor@ruledoctor -g -a claude-code -y
```

Codex 用户把 `claude-code` 改成 `codex`，Cursor 用户改成 `cursor`。

装完一定要新开对话。

**置顶评论：** GitHub 搜 `syf2211/ruledoctor`。你现在项目规则放在 `CLAUDE.md` 还是 `.cursorrules`？

## 2. 教程贴：Cursor rules 不读

**标题：** Cursor 的 .cursorrules 像没写？我现在这样处理

**正文：**

`.cursorrules` 最大的问题不是写得不够细，而是你不知道 Agent 这次到底读没读。

我的做法：

1. 项目根保留 `.cursorrules` / `AGENTS.md`
2. 深层规范写进 `.ruledoctor.json` 的 `required_reads`
3. 新对话先看 Agent 是否报“已读文件 + 硬约束”
4. 危险操作用可选 Hook 拦截

RuleDoctor 是一个开源 Agent Skill，不替你写规则，只负责让 Agent 先读规则。

安装：

```text
npx skills add syf2211/ruledoctor@ruledoctor -g -a cursor -y
```

## 3. 对比贴：写更长 prompt 没用

**标题：** AI 编程不听规则，不一定是 prompt 不够长

**正文：**

我之前也一直加 prompt：更详细、更强硬、更多“必须”。

后来发现问题不是 prompt 长度，而是：

- 规则文件没被读
- 上下文压缩后忘了
- 深层文档从没进上下文
- Agent 嘴上说遵守，实际命令/代码没照做

RuleDoctor 的思路是把“先读规则”变成 Skill 的默认动作。

它不会承诺 100% 管住模型，但会把这件事变成可观察：

- 开场读哪些文件
- 汇报哪几条硬规矩
- 可选 CLI 做事后报告
- 可选 Hook 拦危险 shell

安装：

```text
npx skills add syf2211/ruledoctor@ruledoctor -g -a codex -y
```

**置顶评论：** GitHub 搜 `syf2211/ruledoctor`。如果你有“规则不生效”的例子，可以丢 Issue。

## 5. 英文生态短帖

**Title:** I made a Skill that makes agents read project rules before editing

**Body:**

RuleDoctor is an open Agent Skill for Claude Code, Codex, and Cursor.

It makes the agent read `CLAUDE.md`, `.cursorrules`, `AGENTS.md`, and explicit `required_reads` before touching code.

It is intentionally honest:

- Skill = read rules + concise constraints
- CLI = optional after-the-fact audit
- Hook = optional shell guard for dangerous commands

Install for Codex:

```bash
npx skills add syf2211/ruledoctor@ruledoctor -g -a codex -y
```
