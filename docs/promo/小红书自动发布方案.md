# 小红书自动发布：工具链调研与接入方案

> 目标：在 Cursor / Claude Code 里尽量自动化发小红书图文，**你负责登录与最终确认**，Agent 负责填文案、选图、调用工具。

---

## 1. 结论先说

| 方式 | 可行性 | 说明 |
|------|--------|------|
| 官方开放 API 发帖 | ❌  practically 无 | 个人创作者无稳定公开发帖 API |
| **小红书 MCP**（本地服务 + 浏览器/签名） | ⚠️ 可用但不稳 | 社区项目多，依赖登录态、逆向、风控 |
| **Playwright / Puppeteer 纯网页自动化** | ⚠️ 可用 | 创作者中心 DOM 常变；登录、验证码需人工 |
| **browser-use / Cursor IDE Browser** | ⚠️ 半自动 | 适合「Agent 填表 + 你点发布」 |
| 完全无人值守发帖 | ❌ 不建议 | 封号、验证码、内容审核 |

**推荐路径（可执行）：**  
文案与图片在仓库 `docs/promo/` 准备好 → **优先试 [xpzouying/xiaohongshu-mcp](https://github.com/xpzouying/xiaohongshu-mcp)** 接入 Cursor MCP → 发帖前 **你扫码/确认** → Agent 调用 `publish` 类工具上传轮播图 + 标题正文。

### 1.1 本轮复核（2026-06-18）

GitHub 搜索显示可用候选很多，但适合“帮我发一条图文笔记”的优先级如下：

| 优先级 | 项目 | 本轮观察 | 判断 |
|:--:|------|----------|------|
| P0 | [xpzouying/xiaohongshu-mcp](https://github.com/xpzouying/xiaohongshu-mcp) | Go 项目，约 1.4w stars，更新到 2026-06-18；README 明确支持登录、图文发布、本地图片绝对路径、视频、搜索、评论等 | 首选；适合真实发布，但要处理登录和平台风控 |
| P1 | [autoclaw-cc/xiaohongshu-mcp-skills](https://github.com/autoclaw-cc/xiaohongshu-mcp-skills) | 基于 `xpzouying/xiaohongshu-mcp` 的 Skill 集合，包含 `post-to-xhs` | 可参考工作流；仍需要先部署 MCP |
| P1 | [iFurySt/RedNote-MCP](https://github.com/iFurySt/RedNote-MCP) | npm 包，登录和搜索/读取能力清晰 | 偏读取/搜索，不是本次首选发布工具 |
| P2 | 其他 `xhs-mcp` / `rednote-mcp` 项目 | 有多账号、发布、采集等宣称，但体量与稳定性参差 | 只作备选，避免主号直接测试 |

Codex 本机也支持外部 MCP：`codex mcp add <name> (--url <URL> | -- <COMMAND>...)`。当前本机已启用的 MCP 里**没有**小红书工具；如果选 `xpzouying/xiaohongshu-mcp`，需要先把它按 README 跑起来，再用 `codex mcp add` 加到 Codex，通常新开 Codex 会话后才会暴露工具。

在不想马上接第三方 MCP 的情况下，本次首发更稳的路径是：**Chrome 网页半自动**。Agent 负责上传图片、填标题正文；你负责扫码/验证码/实名/最终发布确认。

## 3. Playwright / browser-use 方案

| 工具 | 作用 |
|------|------|
| 仓库已有 `playwright` devDep | `docs/promos/export-png.mjs` 已用于**导出宣传图** |
| Cursor `cursor-ide-browser` MCP | Agent 可打开创作者中心、上传图片、粘贴文案 |

**流程示例：**

1. 你本机浏览器登录小红书创作者中心（或 MCP 完成扫码登录一次）。  
2. Agent 打开「发布笔记」页 → 上传 `docs/promo/xhs/slide-01.png` … `07.png`。  
3. Agent 从 `docs/promo/copy/平台文案.md` 粘贴标题与正文。  
4. **你人工检查封面、话题、敏感词 → 点击发布。**

**风险：** 选择器变更、滑块验证、频控、账号异常检测。

## 4. 接入 Cursor / Claude Code

### 4.1 Cursor

1. 按 xiaohongshu-mcp 文档 **本地启动 MCP Server**（常见为 Docker 或二进制 + 环境变量）。  
2. **Settings → MCP → Add server**，填入 command/URL（以该项目 README 为准）。  
3. 新建 Agent 对话，附上本地图片路径与 `docs/promo/copy/平台文案.md` 中小红书一节。  
4. 让 Agent 调用 MCP 的发布工具；**首次登录按 MCP 提示扫码**。

### 4.2 Claude Code

- 在 `~/.claude.json` 或项目 MCP 配置中增加同一 MCP server。  
- 与 Cursor 类似：Agent 调工具，你确认登录与发布。

### 4.3 无 MCP 时（仅浏览器 MCP）

```
请用浏览器打开小红书创作者中心发布页，上传以下 7 张图（按顺序），标题和正文使用我提供的 markdown。不要点击最终发布，等我确认。
```

### 4.4 Codex

Codex CLI 支持添加外部 MCP：

```bash
codex mcp add xiaohongshu --url http://127.0.0.1:<port>/mcp
# 或 stdio 形态：
codex mcp add xiaohongshu -- /absolute/path/to/xiaohongshu-mcp
```

具体用 `--url` 还是 stdio 命令，以所选 MCP 项目的 README 为准。添加后建议新开 Codex 会话，再检查工具是否出现。若 MCP 不稳定，直接用 Chrome 发布页半自动完成首发。

## 5. 分工：你 vs Agent

| 步骤 | 谁做 |
|------|------|
| 维护宣传 HTML/PNG、文案 | Agent 可生成；你审核 |
| 小红书账号登录 / 扫码 | **你** |
| 短信/滑块/人脸验证 | **你** |
| 上传图片、填标题正文、选话题 | Agent（MCP 或浏览器） |
| 合规检查、是否点击「发布」 | **建议你** 至少前几次全人工确认 |
| 评论互动、多号矩阵 | 可选；风控自担 |

## 6. 风险清单

- **账号风控**：异常登录、高频发布、营销话术重复 → 限流或封号。  
- **第三方 MCP**：非官方，可能涉及 Cookie/签名逆向；勿把主号密码交给不可信服务。  
- **内容审核**：AI 文案需人工过一遍违禁词。  
- **稳定性**：页面改版导致自动化失效，需维护脚本/MCP 版本。  
- **法律与 ToS**：自动化可能违反平台条款；建议小号测试。
