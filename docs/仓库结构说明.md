# 仓库结构说明

本仓库是 **「可发布的 Skill 包 + 可选 CLI/Hook 工具」** 的 monorepo，不是纯 npm 库，也不是纯文档站。

---

## 用户视角：你只需要关心什么？

```text
skills/ruledoctor/
  SKILL.md          ← Agent 读的唯一必需文件
  README.md         ← 人类安装说明（从 GitHub 进仓库时先看这个或根 README）
```

复制 `skills/ruledoctor` 到 `~/.claude/skills/`（或 Codex/Cursor 对应目录）即可。**不必** clone 整个仓库，除非你要 CLI/Hook。

根目录 [README.md](../README.md) 与 [docs/用户指南.md](用户指南.md) 面向 **装 Skill 的人**；Hook/CLI 标为可选。

---

## 目录该不该在「最终仓库」里？

| 路径 | 是否保留 | 给谁 | 说明 |
|------|:--------:|------|------|
| `skills/` | ✅ | **所有用户** | 标准 skill 仓库入口；CC Switch 子目录填 `skills` |
| `docs/用户指南.md` 等公开文档 | ✅ | 用户 | 安装、生效、Hook 人话 |
| `docs/Hook是什么.md` | ✅ | 用户 | Hook 机制与抽象规则边界 |
| `src/` | ✅ | 维护者 / 要 CLI 的人 | TypeScript CLI；**不会** merge 进 SKILL.md |
| `dist/` | 发布 npm 时 | CLI 用户 | `npm run build` 产物；git 可忽略或随 release |
| `scripts/` | ✅ | 要 Hook 的人 | `setup` 写入的配置指向这些脚本 |
| `plugin/` | ✅ | Claude 插件用户 | 官方插件布局 + 同步的 scripts/skills |
| `test/` | ✅ | CI / 维护者 | 保证 guard/reinject 不破；Skill 安装者无需跑 |
| `examples/` | ✅ | 文档演示 | demo 项目与样例 session |
| `docs/internal/` | ✅ | 维护者 | 真实会话扫描、个案，不冒充用户文档 |
| `AGENTS.md` | ✅ | OpenSkills / 自动化 | 仓库级 Agent 指引 |

**不打算做的：** 把 CLI 源码塞进 `SKILL.md`（会让 Skill 臃肿且 Agent 不需要读实现）；把 `test/` 藏到另一分支（会降低开源可维护性）。

---

## 和「纯 Skill 仓库」（如 PPTMaster）的对比

| 纯 Skill 仓库 | ruledoctor |
|---------------|------------|
| 根 README ≈ 安装 + 用法 | 根 README **以 Skill 为主**，CLI/Hook 折叠为可选 |
| 核心只有 `SKILL.md` | `skills/ruledoctor/SKILL.md` 同样是核心；多出来的 `src/` 是 **同一品牌的可选工具** |
| 无 test | 有 test，因为 Hook 涉及 **真实拦截**，需要回归 |

若你 **只传播 Skill**：分享链接时优先  
`https://github.com/syf2211/ruledoctor/tree/main/skills/ruledoctor`  
或 `npx skills add syf2211/ruledoctor --skill ruledoctor`。
